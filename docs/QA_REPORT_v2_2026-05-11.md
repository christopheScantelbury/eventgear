## ❌ Falhas Críticas

### [4.3] Seção "Fotos" ausente na página de detalhe do material — REGRESSÃO NÃO CORRIGIDA
- **URL:** `/materiais/{id}`
- **Esperado:** Seção "Fotos" com botão "Adicionar foto", upload de JPG, foto principal com badge ⭐
- **Aconteceu:** Página exibe apenas QR Code, Informações e "Zona de perigo". Nenhum campo de upload de foto existe. O fix prometido na v2 não foi implementado.
- **API:** Sem endpoint de upload de foto identificado.

---

### [4.6] Filtro por categoria ausente na UI de Materiais
- **URL:** `/materiais`
- **Esperado:** Filtrar por categoria "Áudio" → retornar 2 resultados (Mesa de Som Yamaha + Caixa de Som JBL)
- **Aconteceu:** O campo de busca só pesquisa por nome. Pesquisar "Áudio" retorna "Nenhum material encontrado". A API suporta `?category=Áudio` (retornou 2 itens), mas o frontend não expõe esse filtro.
- **Console:** Sem erros. Problema de ausência de feature no frontend.

---

### [7.2 / 7.3] Checklist de Saída não exibe itens existentes — REGRESSÃO NÃO CORRIGIDA
- **URL:** `/eventos/cmp17a7wi000dm8d0xrfp3dqv/checklist?tipo=saida`
- **Esperado:** Página lista imediatamente os materiais alocados (Caixa de Som JBL + Cabo XLR 10m). O fix do v2 deveria ter corrigido este bug.
- **Aconteceu:** Página mostra "Checklist não gerado" e botão "Gerar checklist". A API tem 7 itens de checklist existentes (`GET /v1/events/{id}/checklist?tipo=DEPARTURE` retorna 200 com 7 itens). Clicar em "Gerar checklist" não gera nem exibe itens (API retorna 409 — checklist já existe). O loop de v1 persiste.
- **Console:** Sem erros de app; exceções são do chrome-extension.
- **Impacto:** Cenários 7.4 e 7.5 (checkbox manual + progress bar) bloqueados por este bug.

---

### [10.3] Frontend exibe silêncio ao atingir limite de usuários via modal
- **URL:** `/configuracoes/usuarios`
- **Esperado:** Ao tentar criar 4º usuário via UI, mensagem de erro clara com link para `/planos`
- **Aconteceu:** Modal "Novo usuário" abre normalmente. Após preencher e submeter, o modal permanece aberto sem nenhuma mensagem de erro visível (React state issue — o submit real não dispara). Via API direta, o 403 com mensagem "Limite do trial atingido. Escolha um plano para criar mais." retorna corretamente. O frontend não propaga o erro para o usuário.

## ⚠️ Observações

### [PC3] Service Worker: erro de redirect no Vercel
- Console na landing page: `SecurityError: Failed to register a ServiceWorker... The script resource is behind a redirect`. Não impede o funcionamento (SW está `activated` nas páginas da app), mas indica configuração Vercel para `sw.js` incorreta.

### [1.3] Mensagem de erro de login em inglês
- Ao tentar login com senha errada, a mensagem exibida é **"Invalid credentials"** (em inglês). A interface da aplicação é 100% em português. Esperado: "Credenciais inválidas" ou equivalente. Funcional, mas inconsistente.

### [2.1] Toast de "Nome atualizado" não capturável por automação
- A atualização de nome funciona (refletida imediatamente no sidebar e cabeçalho), mas o toast desaparece antes da captura. Comportamento esperado do toast rápido — funcional.

### [8.2] Estado de trial expirado: "Sem plano" ao invés de banner vermelho
- A v2 especifica um "banner vermelho 'Seu acesso está limitado'" para trial expirado. O que aparece é texto "Sem plano" em vermelho + "Período de teste expirado — escolha um plano abaixo". Não há banner vermelho separado no topo. A informação está presente mas no layout diferente do esperado.

### [8.3] Barra de Usuários em vermelho (3/3 = 100%)
- USUÁRIOS: 3/3 com barra vermelha cheia. Visualmente alarmante para o usuário. Funcional e correto.

### [9.1] Ícone PWA no browser chrome não verificável via automação
- Não é possível capturar o ícone de instalação PWA da barra de endereço via screenshot de página. SW está activated, manifest válido. A instalação real requereria interação manual.
