05 Botões
Hierarquia de ações
Primário (âmbar): Ação principal da tela — confirmar saída, criar evento, salvar. Texto dark sobre fundo âmbar. Máximo 1 por tela.
Secundário: Ação alternativa relevante. Ghost: cancelar, voltar. Danger: remoção com confirmação.
Botão especial — Scan QR Code
ou marcar manualmente →
Esse botão é o CTA mais crítico do app. Aparece em destaque na tela de checklist de saída e retorno. Borda âmbar glow, tamanho generoso (48px altura mínima), sempre com ícone de QR à esquerda. Em mobile ocupa 100% da largura.

16 Tom de Voz
Princípios
Direto e operacional
O usuário está no backstage, no escuro, provavelmente com pressa. Linguagem é telegráfica, sem rodeios. Verbos no imperativo.
Preciso nos dados
Nunca "alguns itens" — sempre "3 itens". Nunca "recentemente" — sempre "07 Mai 14:32". Dados de equipamento são dinheiro.
Urgente quando preciso
Itens faltando após evento = alerta vermelho, linguagem de urgência. Estoque OK = verde silencioso. Não alarma à toa.
Técnico sem ser chato
Operadores entendem o gear. Pode usar termos técnicos (Moving Head, Direct Box, truss), mas nunca código de erro sem tradução.

// POST /events/:id/checklist/departure // Confirma item via QR scan ou manualmente async confirmChecklistItem( itemId: string, userId: string, companyId: string, ): Promise<ChecklistItem> { const item = await this.prisma.checklistItem.findFirst({ where: { id: itemId, companyId }, include: { eventMaterial: { include: { material: true } } }, }); if (!item) throw new NotFoundException('Item not found'); if (item.status === 'CONFIRMED') return item; return this.prisma.checklistItem.update({ where: { id: itemId }, data: { status: 'CONFIRMED', confirmedById: userId, scannedAt: new Date(), }, }); }
Fundo: dark-950. Header: nome do contexto em mono à esquerda + "Copiar" em âmbar à direita. Keywords: purple, strings: green, métodos: blue, números: red, comentários: slate-500. Usar apenas em documentação técnica interna, README e Swagger — nunca na UI do operador.
